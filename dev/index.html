ACEmd

ACEmd is molecular dynamics interface for Julia ACE. All MD use cases should use this package.

ACEmd is fully AtomsBase compatable and you should use AtomsBase over older JuLIP. You can still use JuLIP Atoms type as an input, while in a long run JuLIP will be decomissioned.

Example use case

using ACEmd # Reexports AtomsBase and Unitful
using ExtXYZ

# Load potential and atomic structure
fname_ace = joinpath(pkgdir(ACEmd), "data", "TiAl.json")
fname_xyz = joinpath(pkgdir(ACEmd), "data", "TiAl-big.xyz")

data = FastSystem(ExtXYZ.Atoms(read_frame(fname_xyz)))
pot = load_ace_model(fname_ace)

# use ACE to calculate...
ace_energy(pot, data)
ace_forces(pot, data)
ace_virial(pot, data)

Units

ACEmd supports Unitful and it is reexported by default.

Defaults units are

julia> ACEmd.default_energyERROR: UndefVarError: `ACEmd` not defined
julia> ACEmd.default_lengthERROR: UndefVarError: `ACEmd` not defined

Units can be changes when loading potential

pot_ev = load_ace_model(fname_ace;
    energy_unit=u"eV",
    length_unit=u"√Ö",
    cutoff_unit=u"√Ö"
)

ace_energy(pot_ev, data)

Alternatively you can overload the default unit by giving it explicitely

julia> ace_energy(pot_ev, data; energy_unit="hartree")ERROR: UndefVarError: `ace_energy` not defined

Following units are taken into account

  • energy_unit defines the energy unit of the potential
  • length_unit defines the length unit the potential is defined
  • cutoff_unit defines unit for the cutoff the potential is using
